# Design System

Welcome to our design system documentation. This guide will help you understand and use our design system effectively.

## Colors

Our color system is built on a foundation of semantic colors that maintain consistency across light and dark themes.

### Brand Colors
- Primary: Used for main actions and brand identity
- Secondary: Used for supporting elements
- Accent: Used for highlighting and emphasis

### Semantic Colors
- Success: Used for positive states and confirmations
- Warning: Used for cautionary states
- Error: Used for error states and destructive actions
- Info: Used for informational states

## Typography

Our typography system uses a carefully selected set of fonts and sizes to ensure readability and hierarchy.

### Font Families
- Sans: Primary font for UI elements
- Serif: Used for headings and special content
- Mono: Used for code and technical content

### Font Sizes
- xs: 0.75rem (12px)
- sm: 0.875rem (14px)
- base: 1rem (16px)
- lg: 1.125rem (18px)
- xl: 1.25rem (20px)
- 2xl: 1.5rem (24px)
- 3xl: 1.875rem (30px)
- 4xl: 2.25rem (36px)
- 5xl: 3rem (48px)

### Spacing Scale

## Components

Our component library is built on top of Radix UI primitives and styled with Tailwind CSS. Each component is designed to be accessible, customizable, and consistent with our design system.

### Available Components
- Accordion
- Alert
- Avatar
- Badge
- Button
- Card
- Checkbox
- Dialog
- Dropdown Menu
- Form
- Input
- Navigation Menu
- Popover
- Select
- Tabs
- Tooltip

## Usage

To use our design system in your project:

1. Import the required components from `@/components/ui`
2. Use the provided variants and utilities from `@/lib/utils`
3. Follow the component documentation for proper implementation

## Best Practices

1. **Consistency**: Use the provided components and styles to maintain consistency
2. **Accessibility**: All components are built with accessibility in mind
3. **Responsive Design**: Components are designed to work across all screen sizes
4. **Theme Support**: Components support both light and dark themes
5. **Customization**: Use the provided variants and utilities for customization

## Contributing

When contributing to the design system:

1. Follow the established patterns and conventions
2. Ensure components are accessible
3. Add proper documentation and stories
4. Test across different themes and screen sizes
5. Update this documentation when adding new features
